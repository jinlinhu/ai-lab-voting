OpenClaw 完整配置指南

安装流程 · 飞书接入 · 开机自启动 · DeepSeek API 配置

一、安装 OpenClaw

1 安装 Node.js

访问 nodejs.org 下载 LTS 版本（如 v22+），Windows 用户下载 .msi 安装包，一路默认安装即可。

安装完成后打开终端（Win+R → 输入 cmd 或 powershell），验证安装：

node --version
npm --version

2 全局安装 OpenClaw

在终端中执行：

npm install -g openclaw

等待安装完成。如果权限不足（Linux/Mac 可能需要 sudo），Windows 用户请以管理员身份运行终端。

3 启动网关服务

安装完成后，运行以下命令启动 OpenClaw 网关：

openclaw gateway start

首次运行会自动打开配置页面（默认地址 http://localhost:18789）。浏览器打开该地址即可进入 OpenClaw 管理控制台。

4 验证运行状态

在控制台页面中可以看到网关运行状态、日志信息。也可以回到终端执行：

openclaw gateway status

如果显示 running 表示安装成功。

常见问题

  • 端口被占用？启动时指定端口：openclaw gateway start --port 18790
  • 版本升级：npm update -g openclaw
  • 卸载：npm uninstall -g openclaw

二、飞书机器人接入

2.1 创建飞书应用

1 打开飞书开放平台

访问 open.feishu.cn，使用飞书账号登录。

点击右上角「开发者后台」→ 进入「创建应用」→ 选择「企业自建应用」。

2 填写应用信息

输入应用名称（如 "OpenClaw Agent"）、描述，上传应用图标，点击「创建」。

3 获取凭证信息

创建完成后，进入应用详情页 → 左侧菜单「凭证与基础信息」。

这里你会看到两个关键信息：

  • App ID（应用 ID）
  • App Secret（应用密钥）

复制并保存好这两项，一会要用。

2.2 配置机器人能力

1 开启机器人能力

左侧菜单 → 「添加应用能力」→ 「机器人」→ 开启开关。

2 配置事件订阅

左侧菜单 → 「事件与回调」→ 「事件配置」。

添加以下事件：

  • im.message.receive_v1（接收消息事件）

在「请求地址配置」中，填入 OpenClaw 提供的回调地址。Agent 启动后控制台会显示该地址。

3 配置权限

左侧菜单 → 「权限管理」→ 添加以下权限：

  • im:message（消息读写权限）
  • im:chat（群聊信息权限）
  • contact:user.base（用户基本信息）

2.3 在 OpenClaw 中完成绑定

1 进入 OpenClaw 渠道配置

在 OpenClaw 管理控制台中，找到「渠道/Channel」配置页，选择「飞书 / Feishu」。

2 填入凭证

将刚才获取的 App ID 和 App Secret 填入对应字段，保存配置。

3 发布应用

回到飞书开放平台 → 「版本管理与发布」→ 创建新版本 → 填写版本号、更新说明 → 「申请发布」。审核通过后应用即正式可用。

4 添加机器人到群聊

在飞书中创建或打开一个群聊 → 群设置 → 添加机器人 → 搜索你的应用名 → 添加。

现在你可以在群聊中 @机器人 发送消息，测试是否正常响应。

测试技巧

添加机器人后，发送一条消息如「你好」，机器人应该自动回复。如果无响应，检查 OpenClaw 日志和飞书事件回调配置。

三、开机自动启动配置

方案：使用 Windows「启动」文件夹创建快捷方式，简单可靠，无需管理员权限。

1 打开启动文件夹

按 Win + R 打开运行对话框，输入：

shell:startup

点击确定，系统会自动打开「启动」文件夹。

2 创建快捷方式

在打开的文件夹中，右键 → 新建 → 快捷方式。

位置填写（注意替换你的实际用户名）：

C:\Windows\System32\cmd.exe /c "cd /d C:\Users\86132 && openclaw gateway run --port 18789 --daemon"

名称填写：OpenClaw Gateway

3 设置最小化运行

右键刚创建的快捷方式 → 属性 → 运行方式 → 选择「最小化」。

这样每次登录 Windows 时，OpenClaw 网关会在后台自动启动（最小化窗口），不干扰桌面操作。

4 测试验证

重启电脑 → 登录后检查 OpenClaw 是否自动运行：

  • 任务管理器中是否有 node.exe 进程运行（OpenClaw 网关进程）
  • 浏览器访问 http://localhost:18789 能否打开控制台
  • 飞书发送消息给机器人看是否有响应

注意事项

  • 确保 OpenClaw 已正确安装且在命令行中可以运行
  • 如果启动失败，检查快捷方式中的路径和端口号是否正确
  • 快捷方式中的 --daemon 参数让网关以守护进程模式运行

四、DeepSeek API 接入配置

使用 onboard 命令配置模式，交互式填写信息，简单快捷。

4.1 获取 DeepSeek API Key

1 注册 DeepSeek 账号

访问 platform.deepseek.com，注册并登录。

2 创建 API Key

登录后进入控制台 → API Keys → 点击「创建 API Key」。

输入名称（如 "openclaw-agent"），创建后立即复制保存 Key。

注意：API Key 关闭弹窗后不再显示完整内容，务必立即复制保存。

4.2 使用 onboard 命令配置

1 启动 onboard 交互配置

在终端中运行：

openclaw onboard

按提示依次完成各项配置。

2 选择模型供应商

在模型配置步骤中，选择 DeepSeek 作为 AI 模型供应商。

3 填写 API 信息

交互式配置过程中，按提示输入：

  • API Key：粘贴刚创建的 DeepSeek API Key
  • 模型名称：推荐使用 deepseek-chat
  • API 端点：https://api.deepseek.com（默认）
  • 其他参数：Temperature（温度）建议 0.7，Max Tokens 建议 4096

4 测试连接

配置完成后，onboard 会自动进行连接测试。如果测试成功，会显示类似信息：

✓ DeepSeek API 连接成功
  模型: deepseek-chat
  Token 余额: xxx

4.3 手动配置

也可以通过 OpenClaw 管理控制台进行配置：

1 进入控制台

浏览器访问 http://localhost:18789，进入 OpenClaw 管理控制台。

2 配置模型

找到「模型 / Model」或「AI 配置」页面：

  • 添加新模型供应商，选择 DeepSeek
  • 填入 API Key、模型名称、API 端点
  • 保存配置

3 设置默认模型

在「默认模型 / Default Model」中，选择刚配置的 DeepSeek 模型作为默认 AI 模型。

DeepSeek 模型推荐

  • deepseek-chat：通用对话模型，适合日常使用，性价比高
  • deepseek-reasoner：推理增强模型，适合复杂任务
  • API 计费方式：按 Token 计费，可先在 DeepSeek 平台充值

五、端到端验证

1 确认网关运行中

openclaw gateway status

2 在飞书中测试

打开飞书群聊，@机器人 发送一条消息，例如：

@OpenClaw 你好，帮我写一个投票系统

如果机器人正常回复，说明整套配置已完成。

配置完成后的下一步

你现在可以：

  • 在飞书中与 OpenClaw 对话，让 AI 生成网页应用
  • 在 OpenClaw 控制台中添加更多渠道（企业微信、Discord 等）
  • 配置定时任务（Cron Job），让 AI 自动执行周期性工作
  • 安装 Community Skills，扩展 AI 能力
