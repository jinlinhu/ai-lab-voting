# GitHub Pages 一键部署指南

## 🚀 5分钟完成部署

### 第一步：准备文件
1. 将 `GitHub-Pages部署` 文件夹中的所有文件复制到一个新文件夹
2. 确保有以下文件：
   - `index.html` (主程序)
   - `README.md` (说明文档)

### 第二步：创建GitHub仓库
1. 访问：https://github.com/new
2. 填写：
   - Repository name: `ai-lab-voting` (或你喜欢的名字)
   - Description: AI拓边实验室投票系统
   - Public (公开)
   - 不初始化README

### 第三步：上传文件
**方法A：网页上传（最简单）**
1. 进入新建的仓库页面
2. 点击 "Add file" → "Upload files"
3. 拖拽所有文件到上传区域
4. 点击 "Commit changes"

**方法B：使用GitHub Desktop**
1. 下载 GitHub Desktop: https://desktop.github.com
2. 克隆仓库到本地
3. 复制文件到仓库文件夹
4. 提交并推送

### 第四步：开启GitHub Pages
1. 进入仓库 → "Settings"
2. 左侧点击 "Pages"
3. 配置：
   - Source: Deploy from a branch
   - Branch: main (或master)
   - Folder: / (根目录)
4. 点击 "Save"

### 第五步：获取链接
等待约1分钟，刷新页面，看到：
```
Your site is live at https://[你的GitHub用户名].github.io/ai-lab-voting/
```

## 🆘 常见问题

### Q1：页面显示404
- 等待1-2分钟再刷新
- 检查文件是否在根目录
- 确认GitHub Pages已开启

### Q2：功能不正常
- 检查浏览器控制台(F12)有无错误
- 确认Supabase连接正常
- 清除浏览器缓存

### Q3：访问速度慢
- GitHub Pages有全球CDN，通常速度良好
- 可以尝试Cloudflare CDN加速
